1. მაკროსები, მაკროსის ჩაწერა და გაშვება

მაკროსი არის პროგრამა, რომელიც მაკრორეკორდერის საშუალებით მიიღება. მაკროსის ჩაწერის საფუძველზე შესაძლებელია მომხმარებლის მიერ შესრულებული მოქმედებათა თანმიმდევრობით ჩაწერა და შესაბამისი VBA პროგრამული კოდის მიღება. მისი ძირითადი დანიშნულებაა მომხმარებლის მიერ შესრულებული სამუშაოს ავტომატიზაცია; შესაძლებელია ასევე მაკროსის კოდის შემდგომი დამუშავება და სრულყოფა. ყველა მაკროსს აქვს თავისი ინდივიდუალური სახელი; მაკროსის გაშვების შემდეგ კი ავტომატურად შესრულდება ის დანართი, რომელიც შეიქმნა მაკროსის ჩაწერისას. მაკროსის ჩაწერა ხდება Developer მენიუდან Record Macro-ის ბრძანებით. ამ ბრძანების გაშვებისას, მივიღებთ დიალოგურ ფანჯარას სადაც ვანიჭებთ მაკროსს სახელს და სურვილისამებრ შორთქათს (Shortcut). დათანხმების შემდეგ, სრულდება მაკროსის ჩასაწერი მოქმედება - მოქმედების შესრულების შემდეგ აუცილებელია მაკროსის დასრულება - Stop Recording-ის ღილაკით. მნიშვნელოვანია, მაკროსის ჩაწერისას, ყველა ჩვენი მოქმედება იყოს ზუსტი და ზედმეტი მოქმედება არ გავუშვათ პროგრამის გადატვირთვის თავიდან ასარიდებლად. მაკროსის გაშვებისთვის, საჭიროა გააქტიურდეს Macros ბრძანება, დიალოგურ ფანჯარაში იქნეს არჩეული სასურველი მაკროსი და გააქტიურება “Run” ღილაკით. ასევე, აღსანიშნავია, რომ შესაძლებელია მაკროსის გაშვება ღილაკის(Button) შექმნით და ღილაკისთვის სასურველი მაკროსის მინიჭებით. შემდგომ, ღილაკზე ყოველ დაკლიკებაზე გაიშვება მაკროსი.

2. პროექტში ფორმის დამატება, მუშაობა მართვის ელემენტებზე.

UserForm, ანუ სამომხმარებლო ფორმა VBA ენის ერთ-ერთ ყველაზე ძირითად ობიექტს წარმოადგენს. ეკრანული ფორმა, ანუ სამომხმარელო ფორმა – UserForm არის ფანჯარა, რომელიც წარმოადგენს დანართის ინტერფეისის ნაწილს. სამომხმარებლო ფორმასთან მუშაობა მოიცავს რამდენიმე ეტაპს: • სამუშაო ფაილის გახსნა ან შექმნა; • Visual Basic რედაქტორში გადასვლა; • სამომხმარებლო ფორმის შექმნა; • შექმნილი ფორმისათვის თვისებების გამოყენება; • ფორმაზე მართვის ელემენტების შექმნა; • დაიწეროს ხდომილებების დამუშავების პროცედურები. პროექტში ფორმის დამატება დამოკიდებულია იმაზე, თუ სად გვინდა დავამატოთ ფორმა: Access ფაილში, თუ Excel ფაილში. Access ფაილში ფორმის დამატებისას, გადავდივართ: Design მენიუში უნდა ავირჩიოთ Form Design, სადაც ვარედაქტირებთ ჩვენს ფორმას, ვამატებთ განსხვავებულ მართვის ელემენტებს და შემდგომ შენახვისთვის, ფორმას ვაკლიკებთ მარჯვენა ღილაკით და ვირჩევთ “Save”, მას შემდეგ კი ვწერთ ფორმის სახელს. Excel ფაილში კი, ფორმას ვქმნით უშუალოდ Visual Basic-ში; Insert მენიუში ვირჩევთ UserForm ბრძანებას და გამოჩნდება სამომხმარებლო ფორმის რედაქტირების ფანჯარა. შექმნილ ეკრანულ ფორმას default-ად აქვს სათაური UserForm1(ან იმის მიხედვით, თუ მერამდენე ფორმაა). ფორმის შექმნის შემდეგ საჭიროა ამ ობიექტის თვისებების დათვალიერება და ცვლილებების შეტანა, თუ ამას მოითხოვს ამოცანა. თვისებების ფანჯრის ასახვისათვის უნდა ავირჩიოთ ბრძანება: View | Properties Window, ან დავაწკაპუნოთ ინსტრუმენტების პანელზე ღილაკზე. Excel-ში ToolBox-ის ფანჯარით, ხოლო Access ში Design მენიუს ზედა ნაწილში განთავსებული ღილაკებით შესაძლებელია ე.წ მართვის ელემენტების დამატება ფორმაში. Visual Basic -ი გვთავაზობს შემდეგ მართვის ელემენტებს: TextBox ტექსტის შესაყვანი ველი Label ლეიბლი, წარწერა, სათაური Button (CommandButton) ღილაკი, მართვის ღილაკი Tab Control ჩანართები, ტაბების დამატება Link ლინკი Combo Box ველი სიით Toggle Button გადამრთველი/გამომრთველი List Box სია Check Box/Option Button სხვადასხვა ტიპის ღილაკები File Attachment ფაილის მიმაგრება Image სურათის დამატება და მრავალი სხვა თითოეულ მართვის ელემენტს საკუთარი მიზნობრიობა და დიზაინი გააჩნია, ისეთ ღილაკებს კი, როგორიცაა მაგალითად Button/CommandButton გააჩნია უამრავი ფუნქციონალი, მაგალითად, დაკლიკებისას კონკრეტული მაკროსის, ან ხელით გაწერილი კოდის გაშვება და ა.შ

3. ცვლადები, გამოსახულებები.

ცვლადები და გამოსახულებები წარმოადგენენ VBA ენის ძირითად ელემენტებს. ცვლადი, არის მონაცემი, რომლის მნიშვნელობა იცვლება პროგრამის მსვლელობის პროცესში. ცვლადის გამოცხადებისთვის (ინიციალიზებისთვის), საჭიროა მისთვის სახელისა და ტიპის მინიჭება. სახელი არ უნდა შეიცავდეს წერტილს, Space-ს და სპეც.სიმბოლოებს, ასევე სიგრძე არ უნდა აღემატებოდეს 255 სიმბოლოს. ცვლადის სახელი არ უნდა ემთხვეოდეს ჩაშენებულ ფუნქციებს და პროცედურებს. ცვლადების ტიპებიდან ძირითადი შემდეგია: Byte (დიაპაზონი: 0-255); Boolean (True/False); Integer - მთელ რიხვთა ტიპი, რომელიც სულ იტევს 2^32-მდე რიცხვებს. Long - გრძელი მთელ რიცხვთა ტიპი - სულ იტევს 2^64-მდე რიცხვებს. Single - მცოცავ მძიმიანი რიცხვი, იტევს საკუთარ თავში 32 ბიტიან რიცხვებს მცოცავი მძიმით. Double – Single-ს გაგრძელებული ვერსია, მცოცავ მძიმიანი რიცხვი ორმაგი სიზუსტით იტევს 64 ბიტს. Currency - ფულადი მონაცემების ტიპი, იტევს 64 ბიტს - 15 ციფრს მძიმის მარცხნივ და 4-ს მძიმის მარჯვნივ. Date - თარიღისა და დროის აღმნიშვნელი ცვლადი. String - სტრიქონი, მონაცემთა ტიპი, რომელიც ინახავს ტექსტს. Variant - ვარიანტი, ტიპი, რომლის მნიშვნელობაც დამოკიდებულია შემავალ მონაცემებზე. გამოსახულება შედგება კონსტანტების, ცვლადების, ჩადგმული ფუნქციების და არითმეტიკული ოპერაციის ნიშნებისგან. გამოსახულება შეიძლება იყოს მხოლოდ ერთი ელემენტი, ანუ კონსტანტა, ცვლადი ან მიმართვა ჩადგმულ ფუნქციაზე. გამოსახულებების მაგალითებია: (a+b)^2, 45, sin(y), Z და სხვა.

4. პირობითი ოპერატორები

მმართველი სტრუქტურები (ოპერატორები) განსაზღვრავენ პროგრამის შესრულების მიმდევრობას. მათ გარეშე პროგრამის ყველა ოპერატორი შესრულდება მარცხნიდან მარჯვნივ და ზემოდან ქვემოთ. ზოგჯერ მოითხოვება ინსტრუქციების ზოგიერთი ნაკადების მრავალჯერადი შესრულება ან ამოცანის ამოხსნა სხვაგვარად, პარამეტრების ან ცვლადების მნიშვნელობების მიხედვით. ამისთვის, გამოიტენება მართვის ოპერატორები, რომლებიც შემდეგნაირია: If … Then Select Case If … Then If … Then ოპერატორი ამოწმებს ლოგიკურ პირობას და თუ ის ჭეშმარიტია, მაშინ სრულდება Then - ის შემდგომ მოცემული კოდის ბლოკი. ძირითადად, მოცემული ოპერატორის სინტაქსია: If პირობა Then {კოდის ბლოკი} End If End If - ის მითითება აუცილებელია, რათა მოხდეს იმის დადგენა, თუ სად მთავრდება მოცემული ოპერატორი. If … Then ოპერატორში შესაძლებელია დამატებითი პირობის ჩამატება : “Else” - ანუ, წინააღმდეგ შემთხვევაში. If პირობა Then {კოდის ბლოკი 1} Else {კოდის ბლოკი 2} End If შესაბამისად, თუ პირობა ჭეშმარიტია, სრულდება კოდის ბლოკი 1, თუ მცდარი - კოდის ბლოკი 2. შესაძლებელია ElseIf ბლოკის დამატება, რომელიც დამატებით პირობას მიანიჭებს ოპერატორს: If პირობა Then {კოდის ბლოკი 1} ElseIf პირობა Then {კოდის ბლოკი 2} Else {კოდის ბლოკი 3} End If Select Case ოპერატორი წარმოადგენს If … Then … Else ოპერატორის ალტერნატივას. გვაძლევს საშუალებას ვარიანტების სიიდან ამოვარჩიოთ საჭირო მოქმედება. სინტაქსი: Select Case [შესამოწმებელი_გამოსახულება] Case [გამოსახულების_სია 1] {კოდის ბლოკი 1} Case [გამოსახულების_სია 2] {კოდის ბლოკი 2} Case [გამოსახულების_სია 3] {კოდის ბლოკი 3} Else {კოდის ბლოკი 4} End Select

5. მაკროსების სამუშაო გარემო, მოდულები, პროცედურები, ფუნქციები

მაკროსის კოდის შემოწმება, რედაქტირება ან დამოუკიდებლად მაკროსის შექმნა შესაძლებელია Visual Basic რედაქტორის გამოყენებით, რომელიც ჩაშენებულია Microsoft Office-ში . VBA რედაქტორში აღმოსაჩენად, გადავდივართ Developer მენიუში და ვაკლიკებთ “Visual Basic”-ს. რედაქტორის ფანჯარა შედგება შემდეგი კომპონენტებისგან: ინსტრუმენტების პანელი, პროექტის ფანჯარა, თვისებების ფანჯარა, კოდის რედაქტირების ფანჯარა, მენიუ. VBA-ს კოდი განთავსებულია მოდულში. მოდული შედგენა პროცედურებისგან. მოდულში შემავალი ცალკეული პროცედურა არის ან პროცედურა-ფუნქცია, ან პროცედურა ქვეპროგრამა. შესაბამისად, მოდული არის გამოცხადებების და პროცედურების ერთობლიობა, რომელიც ინახება როგორც ერთი მთლიანი. პროცედურა VBA-ს კოდის ნებისმიერი ერთობლიობაა, რომელიც განიხილება როგორც ერთი მთლიანი. როგორც წესი, პროცედურა შედგება ოპერატორებისგან, რომელიც შეასრულებს ან ამოხსნის რაღაც კონკრეტულ ამოცანას. თითოეულ პროცედურას უნდა ჰქონდეს უნიკალური სახელი, რომელიც არ დაემთხვევა უკვე არსებულ ჩაშენებულ ფუნქციებს. VBA ენაში გარდა უკვე ჩაშენებული ფუნქციებისა, შესაძლებელია სამომხმარებლო ფუნქციის შექმნა - რომელიც მიიღებს რაიმე არგუმენტს (თუმცა არაა აუცილებელი არგუმენტის გადაცემა), და დააბრუნებს პასუხს შემავალი მონაცემების დამუშავების შედეგად. ფუნქცია, ფაქტობრივად, არის გარკვეულ მოქმედებათა თანმიმდევრობა, კოდის ერთობლიობა, რომლის გამოძახება შესაძლებელია ედიტორის ნებისმიერ ნაწილში. ფუნქციის დადებითი მხარე ისაა, რომ არ არის საჭირო ერთი და იმავე კოდის და მიზნის მქონე კოდის განმეორება რამოდენიმეჯერ პროგრამის წერისას. განსხვავება ფუნქციასა და პროცედურას შორის ისაა, რომ ფუნქციის გამოძახება შეიძლება რედაქტორში ნებისმიერ ადგილას და აბრუნებს რაიმე მნიშვნელობას, პროცედურა კი - არა. პროცედურის ან ფუნქციის შესაქმნელად, ჯერ არის შესაქმნელი მოდული, რომლის შექმნა შესაძლებელია შემდეგნაირად: Insert  Module. შემდგომ, ამ მოდულის გახსნის შემდეგ ვამატებთ ან პროცედურას, ან ფუნქციას: Insert  Procedure ან Insert  Function

6. ოპერატორები (არითმეტიკული, შედარების, ლოგიკური)

ოპერატორი არის კოდის ელემენტი, რომელიც რაიმე ოპერაციას ასრულებს ერთ ან რამოდენიმე კოდის ელემენტზე, რომელსაც რაიმე მნიშვნელობა აქვს მინიჭებული. VBA-ში ძირითადად შემდეგი ტიპის ოპერატორებია: არითმეტიკული, ლოგიკური, შედარების ოპერატორები. არითმეტიკული ოპერატორების გამოყენებით, შესაძლებელია არითმეტიკული მოქმედებების შესრულება VBA კოდში. არითმეტიკული ოპერატორებიდან შემდეგია ხელმისაწვდომი VBA-ში: „+“ - რომელიც ამატებს ორ მნიშვნელობას; „-„ - აკლებს ორ მნიშვნელობას; „*“ - ამრავლებს ორ მნიშვნელობას; „/“ - ყოფს პირველ მნიშვნელობას მეორეზე. „%“ - აბრუნებს პირველი მნისვნელობის გაყოფისას მეორეზე დარჩენილ ნაშთს. „^“ - ხარისხში აყვანის ოპერატორია. 2^5 - არის ორი ხუთის ხარისხში. ლოგიკური ოპერატორების გამოყენებით შესაძლებელია ლოგიკური ოპერაციების შესრულება, ისეთი, როგორიცაა: AND - ლოგიკური „და“ ოპერატორი; თუ ორივე პირობა True-ა, მაშინ საბოლოო მნიშვნელობა = True. OR - ლოგიკური „ან“ ოპერატორი. თუ ერთი რომელიმე პირობა ორიდან True-ა, მაშინ საბოლოო მნიშვნელობა = True. NOT - ლოგიკური „არა“ ოპერატორი. გამოიყენება ლოგიკური შედეგის შესაბრუნებლად. თუ პირობა True-ა, დაბრუნდება False. XOR - ლოგიკური „გამომრიცხავი ან“ ოპერატორი. წარმოადგენს NOT და OR ოპერატორის კომბინაციას. თუ ერთი და მხოლოდ ერთი პირობაა True, საბოლოო შედეგია True.

7. ციკლის ოპერატორები

VBA კოდის განმაორების ოპერატორები, იგივე ციკლის ოპერატორები გამოიყენება ერთი ოპერატორის ან ოპერატორთა ბლოკის მრავალჯერ შესასრულებლად. VBA-ში გამოიყოფა ორი სხვადასხვა ციკლის ოპერატორი :While და For. While ციკლის ოპერატორი: Do While (პირობა) {კოდის ბლოკი} Loop კოდის ბლოკი შესრულდება მანამ, სანამ პირობა არის True. როდესაც პირობა გახდება False, მოხდება ციკლიდან გამოსვლა და სრულდება Do While …. Loop ოპერატორის მომდევნო ოპერატორები. For ციკლის ოპერატორი: გამოიყენება ერთი ოპერატორის ან ოპერატორთა ბლოკის მრავალჯერ შესასრულებლად. სინტაქსი: For I = M1 To M2 [Step M3] {კოდის ბლოკი} Next i სადაც M1 არის I ცვლადის საწყისი მნიშვნელობა, M2 – I ცვლადის საბოლოო მნიშვნელობა. მანამ, სანამ I ცვლადის მნიშვნელობა მოცემულ შუალედშია, მოხდება არსებული კოდის ბლოკის შესრულება. Next I -კი გადავდივართ ცვლადის შემდეგ მნიშვნელობაზე. ბიჯის სიდიდე Default-ად არის 1, თუმცა თუ გვინდა სხვა ბიჯის მითითება შეგვიძლია Optional ოპერატორის დამატება: Step M3, სადაც M3 -ით ვუთითებთ სასურველ ბიჯს,

8. რიცხვითი გამოსახულებები (მათემატიკური და ფინანსური ფუნქციები)

გამოსახულება შედგება კონსტანტების, ცვლადების, ჩადგმული ფუნქციების და არითმეტიკული ოპერაციის ნიშნებისგან. კერძო შემთხვევაში გამოსახულება შეიძლება იყოს მხოლოდ ერთი ელემენტი, ანუ კონსტანტა, ცვლადი ან მიმართვა ჩადგმულ ფუნქციაზე. გამოსახულებების მაგალითებია: (a+b)^2, 45, sin(y), Z და სხვა. VBA-ში ადგილი აქვს უამრავ სხვადასხვა ფუნქციას, მათ შორის: ლოგიკური ფუნქცია, ფორმატის გარდაქმნის ფუნქციები, თარიღისა და დროის ფუნქციები, ტიპების შემოწმების ფუნქციები; ფუნქციებს შორის მნიშვნელოვან ადგილს იკავებს რიცხვითი ფუნქციები, ანუ იგივე მათემატიკური ფუნქციები. ესენია, მაგალითად: Cos(x), Exp(x), Log(x), Sin(x), Tan(x), Sqr(x) და სხვა. გარდა ამისა, მათემატიკურ ფუნქციებს ეკუთვნის შემდეგი ფუნქციები VBA პროგრამირების ენაში, ესენია: Abs(x) - აბსოლიტური მნიშვნელობა, ანუ მოდული. აბრუნებს x-ის მნიშვნელობას „-„ ნიშნის გარეშე; Sqr(Number := x) - გვიბრუნებს გადაცემული x-ის კვადრატულ ფესვს და აბრუნებს double ტიპის მნიშვნელობას. Exp(Number := x) - გვიბრუნებს ესქპონენტას x ხარისხად. Log(x) - გამოთვლის x-ის ლოგარითმს. Rnd - გვიბრუნებს შემთხვევით რიცხვს 0-დან 1-მდე. და ტრადიციულად, Sin(x), Cos(x), Tan(x). გარდა ამისა, VBA-ში ადგილი აქვს ისეთ ფუნქციებს, როგორიცაა ფინანსური ფუნქციები, მათ გამოყენებით შეიძლება ამა თუ იმ ფინანსური კოეფიციენტების გამოთვლა კოდის გამოყენებით. ესენია: DDB(cost, salvage, life, period) FV(rate, nper, pmt) Ipmt ( rate, per, nper, pv) IRR (values()) MIRR (values(), finance_rate, reinvest_rate) Nper ( rate, pmt, pv)

9. MS Word-ის დაპროგრამება. მუშაობა დოკუმენტებთან, ტექსტთან, ტექსტის ფორმატირება.

Word სისტემა VBA-სთან ერთად გვთავაზობს ტექსტებთან და ტექსტის ფრაგმენტებთან მუშაობის ფართო შესაძლებლობებს. MS Word-ის დაპროგრამებისთვის, პირველ რიგში, როგორც სხვა ფაილებთან მუშაობის დროს, საჭიროა Developer Mode-ს ჩართვა. ეს შესაძლებელია ფაილის Option-ში, Add-In-ში Developer Mode-ს დამატებით. შესაბამისად, Developer მენიუში ვირჩევთ Visual Basic-ს და გადავდივართ შესაბამის კოდის რედაქტორში. დოკუმენტის ამა თუ იმ ფრაგმენტზე წვდომისათვის გამოიყენება ისეთი ობიექტები, როგორიცაა Sections (განყოფილებები), Paragraphs (აბზაცები), Sentences (წინადადებები), Words (სიტყვები) და ა.შ მაგალითად, Words(6) დოკუმენტში მეექვსე სიტყვაზე გადასვლის საშუალებას იძლევა. Sentences(11) - მე-11 წინადადებაზე გადასვლაა. დოკუმენტის განსაზღვრულ ნაწილზე. გადასვლის შემდეგ შეიძლება Font ობიექტის გამოყენება, რომელიც შრიფტის თვისების შეცვლის საშუალებას იძლევა. ეს ობიექტი შეიცავს მეთოდებს, რომელიც მდებარეობს Home|Font –ში. Range ობიექტი არის საბაზო რედაქტირების ოპერაციაში. იგი შეიძლება განვიხილოთ როგორც ტექსტის უწყვეტი ფრაგმენტი, რომელიც არ არის დამოკიდებული კურსორის მდებარეობაზე. Range მოიცავს ისეთ ობიექტებს, როგორიცაა Words (სიტყვები) ან Sentences (წინადადებები), მათზე მიმართვისას თვით Range ობიექტის მითითება არ არის საჭირო, მაგრამ Sections (განყოფილებები) და Paragraphs (აბზაცები) ობიექტებისათვის ეს აუცილებელია.     Cells(n, 1) = i
    sum = sum + i
    n = n + 1
Next i

End Sub

Public Sub getSum()
Dim n As Integer, sum As Integer, num As Integer, namravli As Long

n = Cells(2, 2).Value
num = 0
namravli = 1

For i = 1 To n
    num = num + 5
    namravli = namravli * num

Next

End Sub
